Git & Github Notes

  • cd = change directory:
    • cd "a code"
  • ls = list files and directories
  • ls -a = shows HIDDEN files and directories
  • git init = initializes a new Git repository
  • git status = used to change a file's “Access“, “Modify” and “Change” timestamps to the current time and date, but if the file doesn't exist, the touch command creates it
  • git add = adds a file to the staging area, ready to be committed:
    • git add index.html style.css
    • git add . = adds all changes to staging area
  • git rm --cached = removes file from staging area, aka unstage
  • git rm = removes
  • cat = allows us to create single or multiple files, view content of a file, concatenate files and redirect output in terminal or files
  • git log = commit history
  • git clone = clones a remote repo to local machine (from Github to PC), That's called PULLING
  • git commit -m "describe what u did"
  • If u add another m- (message) it'll be a description
  • git push = pushes the changes to live repo
  • go to github > settings > SSH and GPG keys > New ssh key > title and key > add
  • This is helpful:
    • https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent
  • Remember:
    1. Create a github repo
    2. Clone it locally
    3. Move your files there
    4. Push it on the remote repo
  • git push -u origin main
    • Do tihs the first time, -u will make it so that u don't have to type "origin main" every time u push from now on
